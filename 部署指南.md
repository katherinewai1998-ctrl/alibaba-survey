## 📋 部署步骤

### 第 1 步：推送到 GitHub

1. **在 GitHub 创建新仓库**
   - 访问 https://github.com
   - 点击右上角 "+" → "New repository"
   - 仓库名称：`alibaba-survey`（或其他名称）
   - 设为 **Public** 或 **Private** 都可以
   - 点击 "Create repository"

2. **推送代码到 GitHub**

在终端执行以下命令（替换 `YOUR_USERNAME` 为您的 GitHub 用户名）：

```bash
cd /Users/kkkat/.easyclaw/workspace/alibaba-survey

# 设置 Git 用户信息（首次使用需要）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 添加远程仓库（替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/alibaba-survey.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 第 2 步：部署到 Render

1. **访问 Render 并登录**
   - 网址：https://render.com
   - 点击 "Get Started for Free"
   - 使用 GitHub 账号登录（推荐）或邮箱注册

2. **创建 Web Service**
   - 登录后，点击 "New +" → "Web Service"
   - 点击 "Connect a repository"
   - 找到并选择 `alibaba-survey` 仓库
   - 点击 "Connect"

3. **配置服务**
   ```
   Name: alibaba-survey（或您喜欢的名称）
   Region: Singapore（新加坡，离中国最近）
   Branch: main
   Root Directory: （留空）
   Runtime: Node
   Build Command: echo "Building..."
   Start Command: node server/index.js
   Instance Type: Free（免费套餐）
   ```

4. **添加环境变量**
   - 点击 "Advanced" 展开
   - 点击 "Add Environment Variable"
   - 添加：
     ```
     Key: NODE_ENV
     Value: production
     ```
   - 再添加：
     ```
     Key: ADMIN_PASSWORD
     Value: admin123（或您自定义的管理密码）
     ```

5. **创建服务**
   - 点击 "Create Web Service"
   - 等待部署完成（约 2-5 分钟）

### 第 3 步：获取公网地址

部署完成后，您会看到：
- **URL**: `https://alibaba-survey.onrender.com`

这就是您的公网访问地址！

## ⚠️ 重要提示

### Render 免费套餐限制
- **每月 750 小时**免费额度（足够 24/30 天运行）
- **15 分钟无访问**会自动休眠
- **下次访问**需要 30-50 秒唤醒时间

### 避免休眠的方法
1. 使用 [UptimeRobot](https://uptimerobot.com) 免费监控服务
   - 注册后添加监控
   - URL: `https://您的应用.onrender.com/health`
   - 设置 5 分钟检查一次
   - 这样应用就不会休眠了

2. 或升级到付费套餐（$7/月）

### 修改管理密码
在 Render Dashboard：
1. 进入您的服务
2. 点击 "Environment" 标签
3. 修改 `ADMIN_PASSWORD` 的值
4. 点击 "Save Changes"
5. 服务会自动重启

## 🆘 遇到问题？

### 部署失败
- 检查 Logs 标签页的错误信息
- 确保 `package.json` 配置正确
- 确保 `server/index.js` 没有语法错误

### 无法访问
- 等待部署完成（显示 "Live" 状态）
- 检查是否休眠（访问 `/health` 端点）
- 查看 Logs 是否有错误
