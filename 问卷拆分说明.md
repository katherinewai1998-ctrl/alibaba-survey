## 🚀 部署方式

### 方式一：分别部署到不同域名

**问卷 A**：
```bash
# 复制 index-cloud-ai.html 为 index.html
cp public/index-cloud-ai.html public/index.html
# 部署到云平台
```
访问地址：`https://cloud-ai-survey.xxx.com`

**问卷 B**：
```bash
# 复制 index-ecommerce-office.html 为 index.html
cp public/index-ecommerce-office.html public/index.html
# 部署到另一个云平台或子域名
```
访问地址：`https://ecommerce-survey.xxx.com`

### 方式二：同一域名，不同路径

修改服务器路由配置：
- `/cloud-ai` → 访问 `index-cloud-ai.html`
- `/ecommerce-office` → 访问 `index-ecommerce-office.html`

### 方式三：提供下载链接

将两个 HTML 文件分别发送给不同调研对象，本地打开填写。

## 📊 数据管理

两份问卷的数据会提交到同一个后端 API（`/api/submissions`），可以通过以下方式区分：

### 方式一：添加问卷类型字段

在每个问卷的 `form` 对象中添加：
```javascript
form: {
  surveyType: 'cloud-ai',  // 或 'ecommerce-office'
  // ... 其他字段
}
```

### 方式二：后台筛选

在后台管理页面，根据提交的数据结构自动识别问卷类型：
- 包含 `cloudProviders`、`qwenUser` 字段 → 问卷 A
- 包含 `taobaoSeller`、`taobaoFlash`、`dingtalkUser` 字段 → 问卷 B
