## 方案一：使用 Serverless Devs 命令行部署（推荐）

### 第 1 步：安装 Serverless Devs

```bash
# 安装 Serverless Devs CLI
npm install -g @serverless-devs/s
```

### 第 2 步：配置阿里云账号

```bash
# 配置阿里云账号信息
s config add

# 按提示输入：
# Account alias: aliyun
# AccessKeyId: [您的 AccessKey ID]
# AccessKeySecret: [您的 AccessKey Secret]
```

**获取 AccessKey：**
1. 访问：https://ram.console.aliyun.com/manage/ak
2. 登录阿里云账号
3. 创建 AccessKey
4. 复制 AccessKey ID 和 Secret

### 第 3 步：部署到 SAE

```bash
cd /Users/kkkat/.easyclaw/workspace/alibaba-survey

# 部署
s deploy
```

### 第 4 步：查看部署结果

```bash
# 查看应用状态
s info

# 查看日志
s logs
```

部署完成后会显示公网访问地址！

## 方案二：使用阿里云控制台部署（无需命令行）

### 第 1 步：登录阿里云

1. 访问：**https://www.aliyun.com**
2. 登录/注册账号
3. 完成实名认证

### 第 2 步：开通 SAE 服务

1. 访问：**https://sae.console.aliyun.com**
2. 点击 **"立即开通"**
3. 按提示完成开通

### 第 3 步：创建应用

1. 点击 **"创建应用"**
2. 填写配置：

#### 基本信息
```
应用名称：alibaba-survey
地域：华东 1（杭州）
命名空间：默认
```

#### 部署方式
```
部署来源：GitHub
仓库地址：https://github.com/katherinewai1998-ctrl/alibaba-survey
分支：main
代码路径：/
```

#### 应用配置
```
运行时：Node.js 18
启动命令：node server/index.js
```

#### 实例配置
```
CPU: 0.25 Core
内存：512 MB
实例数：1
```

#### 环境变量
```
NODE_ENV = production
ADMIN_PASSWORD = admin123
PORT = 8080
```

#### 网络配置
```
开启公网访问：是
负载均衡：SLB（自动创建）
```

### 第 4 步：确认创建

点击 **"确认创建"**，等待 3-5 分钟部署完成。

### 第 5 步：获取访问地址

部署完成后，在应用详情页可以看到：
- **公网地址**：`http://alibaba-survey.xxx.sae.aliyuncs.com`

## 方案三：使用阿里云函数计算 FC（更便宜）

### 创建函数计算服务

1. 访问：**https://fc.console.aliyun.com**
2. 点击 **"创建服务"**
3. 服务名称：`alibaba-survey`

### 创建函数

1. 在服務中点击 **"创建函数"**
2. 选择 **"事件函数"**
3. 配置：
   ```
   函数名称：alibaba-survey
   运行环境：Node.js 18
   代码上传方式：GitHub
   仓库地址：https://github.com/katherinewai1998-ctrl/alibaba-survey
   ```

4. 触发器配置：
   ```
   触发器类型：HTTP
   认证方式：无需认证
   ```

5. 环境变量：
   ```
   NODE_ENV = production
   ADMIN_PASSWORD = admin123
   ```

## 💰 费用对比

### SAE（Serverless App Engine）
- 0.25 Core + 512MB 内存
- 约 **0.5-1 元/天**
- 新用户有免费额度

### 函数计算 FC
- 按调用次数和运行时间计费
- 约 **0.3-0.5 元/天**
- 每月有免费额度

### ECS 云服务器（不推荐）
- 最便宜的实例：约 **30-50 元/月**
- 需要自己维护服务器

## 🔧 修改配置

### 修改管理密码

编辑 `s.yaml` 文件：
```yaml
envs:
  ADMIN_PASSWORD: "您的新密码"
```

然后重新部署：
```bash
s deploy
```

### 修改地域

编辑 `s.yaml`：
```yaml
region: "cn-shenzhen"  # 华南 1（深圳）
```

可选地域：
- `cn-hangzhou` - 华东 1（杭州）
- `cn-shanghai` - 华东 2（上海）
- `cn-shenzhen` - 华南 1（深圳）
- `cn-beijing` - 华北 2（北京）

## 🆘 常见问题

### 1. 部署失败
- 检查 AccessKey 是否正确
- 检查 GitHub 仓库是否公开
- 查看日志：`s logs`

### 2. 无法访问
- 检查是否开启公网访问
- 检查安全组配置
- 等待 3-5 分钟部署完成

### 3. 费用过高
- 调整实例配置（降低 CPU/内存）
- 设置最小实例数为 1
- 使用函数计算 FC 更便宜
